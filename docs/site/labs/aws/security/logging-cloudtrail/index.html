AWS Logging Basics: CloudTrail and Audit Visibility

AWS Security L1 30–45 mins

This lab introduces AWS CloudTrail and explains how logging provides visibility, accountability, and security assurance.

What You’ll Build

  • An AWS account with CloudTrail enabled
  • Visibility into API activity
  • Audit records for user actions
  • Confidence in “who did what”

Zero Trust Mapping — Observability & Audit

This lab directly maps to the Zero Trust principle of Observability. Zero Trust assumes breach and therefore requires continuous visibility into all actions taken across systems.

  • Visibility: CloudTrail records all API activity across the account
  • Accountability: Every action is tied to an identity
  • Auditability: Logs are immutable and time-ordered
  • Detection & Response: Audit logs enable investigations and forensics

Without logging, Zero Trust cannot function. Observability is the foundation that allows trust decisions to be verified rather than assumed.

Prerequisites

  • An AWS account
  • Access to the AWS Management Console
  • No prior logging experience required

Lab Steps

Step 1 — Understand What Logging Is

Objective

Understand why logging exists in cloud security.

Simple Explanation

“Who did what, and when?”

  • You cannot investigate incidents without logs
  • You cannot prove compliance without logs
  • You cannot trust security controls without logs

CloudTrail is AWS’s source of truth for account activity.

Step 2 — Enable CloudTrail

Objective

Turn on CloudTrail to start recording activity.

  1. Open CloudTrail
  2. Click Create trail
  3. Name the trail: stc-cloudtrail-basics
  4. Select All regions
  5. Enable Management events
  6. Create or select an S3 bucket for logs
  7. Create the trail

Step 3 — Generate and View Audit Events

Objective

See real audit records in action.

  1. Create an AWS resource (for example, an S3 bucket)
  2. Return to CloudTrail
  3. Open Event history
  4. Filter by event name or username

This is how security teams investigate incidents.
