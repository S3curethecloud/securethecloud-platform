AWS IAM Basics: Users, Groups, and Permissions

AWS Identity L1

This lab introduces AWS Identity and Access Management (IAM) and teaches how to create users, groups, and permissions safely using best practices.

Overview

In this lab, you will configure foundational IAM components to securely manage access in an AWS account using industry best practices.

Steps

Step 1 — Understand IAM and the Root User

Understand what IAM is and why the root user should not be used for everyday work.

Validation

  • Confirm IAM users do not have root-level permissions
  • Verify least-privilege policies are attached to groups
  • Confirm IAM actions appear in CloudTrail logs

Teardown

  • Remove test IAM users created during the lab
  • Delete IAM groups and policies if no longer needed
  • Ensure root account credentials are secured
