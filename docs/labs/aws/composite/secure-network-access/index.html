Secure Network Access with IAM, Security Groups, and CloudTrail

Learn how to enforce Zero Trust network access by combining IAM identity, least-privilege network controls, and CloudTrail audit evidence.

What You’ll Learn

  • How identity and network controls jointly authorize access
  • How Security Groups and Network ACLs restrict traffic paths
  • How CloudTrail proves access and denial decisions

Overview

In this composite lab, you will design and validate a Zero Trust network access model where identity, network controls, and audit logging must all agree before access is granted.

Steps

  1. Design the Zero Trust model by defining how IAM identity, Security Groups, Network ACLs, and CloudTrail interact to control access.
  2. Create an IAM identity with limited permissions and launch an EC2 instance without public exposure.
  3. Apply Security Group rules that allow access only from approved sources and deny all other traffic by default.
  4. Add Network ACL guardrails to prevent accidental exposure caused by misconfigured Security Groups.
  5. Test allowed and denied access scenarios by changing identity and network conditions.
  6. Review CloudTrail logs to confirm all access attempts and enforcement decisions are auditable.

Validation

  • Access succeeds only when identity and network rules permit it
  • Denied access attempts are visible in CloudTrail
  • No resource is reachable without explicit authorization

Teardown

  • Delete temporary IAM identities or roles
  • Remove Security Group and Network ACL test rules
  • Terminate the EC2 instance if no longer required
