Secure Network Access with IAM, Security Groups, and CloudTrail

AWS Composite L2 60–90 mins

This lab combines identity, network controls, and audit logging to enforce Zero Trust access to cloud resources.

Start Lab →

What You’ll Build

  • An EC2 instance protected by identity and network controls
  • Security Groups enforcing least privilege access
  • Network ACLs providing subnet-level guardrails
  • CloudTrail evidence of access attempts

Prerequisites

  • Completion of L1 IAM, Network, and Logging labs
  • An AWS account
  • Access to the AWS Management Console

Lab Steps

Step 1 — Design the Zero Trust Network Model

Objective

Define how identity and network controls work together.

Key Principles

  • IAM controls who can initiate actions
  • Security Groups control what traffic is allowed
  • Network ACLs act as a safety net
  • CloudTrail records who attempted what

Why This Matters

Network security without identity is weak.
Identity without network controls is incomplete.

Step 2 — Create IAM Identity and EC2 Resource

Actions

  1. Create an IAM user or role with EC2 read access
  2. Ensure no admin permissions are granted
  3. Launch an EC2 instance in a VPC
  4. Do not assign a public IP initially

Why This Matters

Resources should exist before access is granted.

Step 3 — Enforce Access with Security Groups

Actions

  1. Create a Security Group
  2. Allow SSH (port 22) only from your IP address
  3. Deny all other inbound traffic (default behavior)

Outcome

  • Access is explicitly allowed
  • Everything else is denied

Step 4 — Add Network ACL Guardrails

Actions

  1. Create a Network ACL for the subnet
  2. Allow required ports
  3. Explicitly deny unnecessary traffic

Outcome

Misconfigured Security Groups cannot expose the resource.

Step 5 — Attempt and Validate Access

Actions

  • Access from allowed IP → succeeds
  • Access from disallowed IP → fails
  • Remove IAM permissions temporarily → access denied

Outcome

Identity and network controls are both required.

Step 6 — Audit with CloudTrail

Actions

  • Open CloudTrail
  • Review EC2 actions
  • Review Security Group modifications
  • Review failed access attempts

Outcome

Zero Trust is provable, not assumed.
