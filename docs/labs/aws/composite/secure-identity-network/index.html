Secure Identity Federation and Network Access

AWS Composite L2 75–105 mins

This lab combines workforce identity federation with AWS network controls to enforce Zero Trust access based on identity, network boundaries, and audit evidence.

Start Lab →

What You’ll Build

  • Federated workforce access into AWS
  • Permission sets enforcing least privilege
  • Network boundaries restricting access paths
  • Audit trails proving identity- and network-based decisions

Prerequisites

  • Completion of L1 IAM, Network, and Logging labs
  • Completion of AWS ↔ Entra Federation lab
  • An AWS account with IAM Identity Center enabled

Lab Steps

Step 1 — Define Identity & Network Trust Boundaries

Objective

Decide how federated identity and network controls work together.

Key Decisions

  • Identity decides who may attempt access
  • Network decides from where access is allowed
  • Both must agree for access to succeed

Why This Matters

Zero Trust requires multiple independent controls.

Step 2 — Configure Federated Access & Permission Sets

Actions

  1. Use Entra ID (or your IdP) as the identity provider
  2. Map groups to IAM Identity Center permission sets
  3. Grant access to specific AWS accounts only
  4. Avoid direct user permissions

Outcome

  • Access is identity-driven
  • Least privilege is enforced centrally

Step 3 — Apply Network Controls to Federated Resources

Actions

  1. Launch an EC2 instance in a VPC
  2. Attach a Security Group allowing:
    • SSH / HTTPS only from approved IP ranges
  3. Apply a Network ACL as a subnet guardrail

Outcome

Identity alone is insufficient. Network boundaries must also allow traffic.

Step 4 — Attempt Access Scenarios

  • Federated user from allowed IP → Success
  • Federated user from disallowed IP → Denied
  • Non-federated identity → Denied

Outcome: Access requires both identity and network approval.

Step 5 — Validate with CloudTrail

Actions

  • Review CloudTrail events for federated role assumptions
  • Review network-related API calls
  • Review denied access attempts

Outcome: Decisions are auditable and attributable.

Step 6 — Review Zero Trust Posture

Checks

  • No standing credentials
  • No public network exposure
  • All access is time-bound and auditable

Outcome: Zero Trust is enforced end-to-end.
