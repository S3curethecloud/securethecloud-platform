Secure S3 End-to-End with IAM, KMS, Policies, and CloudTrail

Learn how to secure Amazon S3 end-to-end by combining IAM permissions, KMS encryption, bucket policies, and CloudTrail auditing to enforce Zero Trust.

What You’ll Learn

  • How IAM and bucket policies restrict S3 access
  • How KMS enforces encryption and key usage
  • How CloudTrail proves access and enforcement decisions

Overview

In this composite lab, you will build a production-grade S3 bucket that enforces least privilege, mandatory encryption, safe-by-default policies, and auditable access.

Steps

  1. Design the security model by defining how IAM, KMS, S3 policies, and CloudTrail work together before creating any resources.
  2. Create an IAM identity with least-privilege permissions for S3 access and controlled use of a specific KMS key.
  3. Create a customer-managed KMS key and an S3 bucket with SSE-KMS encryption enabled and all public access blocked.
  4. Apply S3 bucket policies that deny unencrypted uploads and prevent public access or unauthorized principals.
  5. Test allowed and denied scenarios to confirm misconfigurations fail safely and intended access succeeds.
  6. Review CloudTrail logs to verify who accessed S3, which KMS key was used, and when actions occurred.

Validation

  • Unencrypted or public access attempts are denied
  • Authorized access succeeds using the correct IAM identity
  • All actions are visible and attributable in CloudTrail

Teardown

  • Delete test IAM identities or detach permissions
  • Remove the S3 bucket and KMS key if no longer required
  • Confirm no residual access paths remain
