AWS Network Security Basics

Learn how AWS network security controls traffic using VPC boundaries, Security Groups, and Network ACLs to enforce least-privilege access.

What You’ll Learn

  • How VPCs establish network isolation in AWS
  • How Security Groups control instance-level traffic
  • How Network ACLs enforce subnet-level rules

Overview

In this lab, you will explore how AWS network security layers work together to control inbound and outbound traffic using VPCs, Security Groups, and Network ACLs.

Steps

Step 1 — Understand Network Boundaries (VPC)

A VPC is your isolated network environment in AWS that defines IP ranges, routing, and connectivity boundaries.

Step 2 — Control Traffic with Security Groups

  1. Go to EC2 → Security Groups
  2. Create a new security group
  3. Allow inbound SSH (port 22) from your IP address
  4. Leave all other inbound traffic denied by default

Step 3 — Add Subnet-Level Protection with Network ACLs

  1. Go to VPC → Network ACLs
  2. Create a Network ACL for your subnet
  3. Add rules to allow required traffic
  4. Explicitly deny unwanted ports

Validation

  • Instances are reachable only on explicitly allowed ports
  • Security Group rules enforce least-privilege access
  • Network ACL rules affect traffic at the subnet level

Teardown

  • Remove test Security Group rules if no longer needed
  • Delete test Network ACLs created for this lab
  • Confirm no unnecessary network access remains open
