Lab Steps

Step 1 — Understand Network Boundaries (VPC)

Objective

Understand what a VPC is and why network boundaries matter.

Simple Explanation

A VPC is your private network inside AWS.

Anything inside the VPC:

  • Is isolated from the internet by default
  • Only allows traffic you explicitly permit

This is the first line of defense in Zero Trust networking.

Step 2 — Control Traffic with Security Groups

Objective

Use Security Groups to allow only required traffic.

Actions

  1. Go to EC2 → Security Groups
  2. Create a new security group
  3. Allow inbound SSH (port 22) from your IP address
  4. Leave all other inbound traffic denied (default behavior)

Why This Matters

Security Groups are stateful firewalls:

  • If traffic is allowed in, return traffic is allowed out
  • Least privilege is enforced by default

Step 3 — Add Subnet-Level Protection with NACLs

Objective

Understand how Network ACLs provide an additional control layer.

Actions

  1. Go to VPC → Network ACLs
  2. Create a Network ACL for your subnet
  3. Add rules to allow required traffic
  4. Explicitly deny unwanted ports

Why This Matters

Network ACLs are stateless:

  • Every direction must be explicitly allowed
  • They act as a guardrail against misconfigured Security Groups
