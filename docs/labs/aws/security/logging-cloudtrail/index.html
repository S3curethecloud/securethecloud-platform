AWS Logging Basics: CloudTrail and Audit Visibility

Learn how AWS CloudTrail records account activity and enables security teams to audit who performed specific actions and when.

What You’ll Learn

  • How AWS CloudTrail captures and stores API activity
  • How to enable CloudTrail across an AWS account
  • How to review audit events for investigation and compliance

Overview

In this lab, you will enable AWS CloudTrail and examine how audit logs record API activity across an AWS account for security and compliance use cases.

Steps

Step 1 — Understand What Logging Is

Logging answers the fundamental security question: who did what, and when.

Step 2 — Enable CloudTrail

  1. Open the CloudTrail console
  2. Create a new trail named stc-cloudtrail-basics
  3. Select all regions
  4. Enable management events
  5. Select or create an S3 bucket for logs
  6. Create the trail

Step 3 — Generate and Review Audit Events

  1. Create a simple AWS resource (for example, an S3 bucket)
  2. Return to CloudTrail
  3. Open Event history
  4. Filter events by service or user

Validation

  • CloudTrail is enabled in all regions
  • API activity appears in Event history
  • Events are associated with specific identities

Teardown

  • Delete the test CloudTrail trail if no longer required
  • Remove any temporary AWS resources created during the lab
