# Lab Steps

## Step 1 — Create a KMS Key
- Create a customer-managed KMS key
- Assign an alias

## Step 2 — Configure Key Policy
- Grant least-privilege access
- Restrict administrative actions

## Step 3 — Use the Key
- Encrypt a test resource (e.g., S3 object or EBS volume)

## Step 4 — Review Key Usage
- Inspect CloudTrail events for KMS activity
