AWS S3 Public Access Block & Bucket Policy Enforcement

AWS Storage L1 25–40 mins

This lab teaches how to prevent accidental public access to S3 buckets using AWS Public Access Block settings and bucket policies.

Start Lab →

What You’ll Build

  • An S3 bucket with public access fully blocked
  • Public Access Block settings applied and verified
  • A bucket policy that denies public access
  • Validation that objects cannot be accessed publicly

Prerequisites

  • An AWS account
  • Access to the AWS Management Console
  • Basic understanding of Amazon S3

Lab Steps

Step 1 — Understand Public Access Risks

Objective

Understand how S3 buckets accidentally become public and why this is a common cause of cloud data breaches.

Simple Explanation

If an S3 bucket is public, anyone on the internet can read the data. No login is required.

Key Concepts

  • Public bucket policies
  • Public ACLs
  • Anonymous access

Step 2 — Enable S3 Public Access Block

Objective

Block all forms of public access at the bucket level.

  1. Open the Amazon S3 console
  2. Select your bucket
  3. Go to the Permissions tab
  4. Open Block public access
  5. Enable all block public access options
  6. Save changes

What This Does

  • Prevents public ACLs
  • Prevents public bucket policies
  • Overrides accidental misconfigurations

Step 3 — Test Public Access (Should Fail)

Objective

Confirm that the bucket is not publicly accessible.

  1. Upload a test file
  2. Copy the object URL
  3. Open the URL in an incognito browser

Expected Result

Access is denied. The file is not readable.

Why This Matters

Security controls must be verified, not assumed.

Step 4 — Enforce Public Access Block at the Account Level

Objective

Prevent public S3 access across the entire AWS account.

  1. Open the Amazon S3 console
  2. Select Block Public Access settings for this account
  3. Enable all block public access options
  4. Save and confirm

What This Does

  • Overrides public ACLs account-wide
  • Overrides public bucket policies
  • Prevents future misconfigurations

Step 5 — Add a Bucket Policy to Explicitly Deny Public Access

Objective

Add an explicit policy that denies any public access attempts.

  1. Open your S3 bucket
  2. Go to Permissions → Bucket policy
  3. Paste a deny policy for public access
  4. Replace YOUR-BUCKET-NAME
  5. Save the policy

What This Does

  • Denies anonymous access
  • Reinforces Public Access Block
  • Provides auditable enforcement

Step 6 — Audit and Prove Public Access Is Blocked

Objective

Verify that public access is fully blocked and produce evidence.

  1. Attempt to access an object using its public URL
  2. Confirm AccessDenied
  3. Verify Public Access Block shows On

End of Step 6 — Lab Complete

  • Public access blocked at bucket and account levels
  • Explicit deny policies in place
  • Audit-ready evidence produced
  • Risk of accidental exposure eliminated
