AWS S3 Encryption Basics

AWS Data L1 30–45 mins

This lab introduces encryption at rest in Amazon S3 and walks through enabling and validating server-side encryption using AWS-managed and KMS-managed keys.

Start Lab →

What You’ll Build

  • An S3 bucket with default encryption enabled
  • Objects encrypted using SSE-S3 and SSE-KMS
  • A basic KMS key used for S3 encryption
  • Verification of encryption at rest

Prerequisites

  • An AWS account with permission to create S3 buckets
  • Basic familiarity with the AWS Console
  • No prior encryption experience required

Lab Steps

Step 1 — Create an S3 Bucket

  1. Open the Amazon S3 console
  2. Create a new bucket with a globally unique name
  3. Leave public access blocked (default)

Step 2 — Enable Default Encryption

  1. Navigate to the bucket’s Properties tab
  2. Enable Default Encryption
  3. Select SSE-S3 as the encryption method

Step 3 — Upload and Verify an Object

  1. Upload a test file to the bucket
  2. Open the object details
  3. Confirm encryption status shows Server-side encryption

Step 4 — Encrypt with SSE-KMS

  1. Create a new KMS key
  2. Upload another object using SSE-KMS
  3. Select your custom KMS key

Step 5 — Compare Encryption Methods

  1. Review object metadata for both files
  2. Note the differences between SSE-S3 and SSE-KMS
