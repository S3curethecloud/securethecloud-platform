AWS ↔ Entra ID Workforce Identity Federation

AWS Entra ID Identity L4 60–90 mins

This lab guides you through implementing production-grade workforce identity federation between AWS IAM Identity Center and Microsoft Entra ID using SAML for authentication and SCIM for authorization.

Start Lab →

What You’ll Build

  • AWS IAM Identity Center integrated with Microsoft Entra ID
  • SAML-based authentication between AWS and Entra
  • SCIM user and group provisioning
  • Role-based access across cloud environments

Prerequisites

  • An AWS account with IAM Identity Center enabled
  • A Microsoft Entra ID tenant with admin permissions
  • Basic understanding of IAM and SAML concepts

Lab Steps

Step 1 — Prepare Identity Systems (Foundation)

Objective

Ensure both identity systems (AWS and Microsoft Entra ID) are ready to participate in federation.

What You Are Establishing

  • Entra ID as the Identity Provider (IdP)
  • AWS IAM Identity Center as the Service Provider (SP)

Actions

  • Confirm AWS IAM Identity Center is enabled in the AWS account
  • Confirm Microsoft Entra ID tenant access with admin permissions
  • Decide which workforce users and groups will be federated

Why This Matters

Federation failures are commonly caused by misaligned identity foundations. This step ensures both systems are prepared before trust is established.

Step 2 — Create the Federation Trust (SAML)

Objective

Establish SAML-based authentication trust between Entra ID and AWS IAM Identity Center.

What You Are Building

  • Entra ID authenticates workforce users
  • AWS trusts Entra’s SAML assertions

Actions

  • Configure an external SAML identity provider in AWS IAM Identity Center
  • Download AWS SAML metadata
  • Create an Enterprise Application in Entra ID
  • Upload AWS metadata and configure SAML claims

Why This Matters

This step defines the cryptographic trust boundary. If SAML trust is incorrect, authentication will fail regardless of downstream configuration.

Step 3 — Enable SCIM Provisioning (Authorization)

Objective

Automate user and group lifecycle management from Entra ID into AWS.

What You Are Building

  • Automated user provisioning
  • Group-to-permission-set mapping
  • Continuous authorization enforcement

Actions

  • Generate a SCIM endpoint and access token in AWS IAM Identity Center
  • Configure SCIM provisioning in Entra ID
  • Enable user and group provisioning
  • Assign Entra groups to the AWS enterprise application

Why This Matters

SAML answers who the user is. SCIM defines what they are allowed to do. Without SCIM, authorization drifts and deprovisioning becomes unreliable.

End of Step 3 — What You Have Achieved

  • Authentication flows from Entra ID to AWS
  • Authorization is group-driven and automated
  • AWS access is centrally governed
  • Manual IAM user management is eliminated
