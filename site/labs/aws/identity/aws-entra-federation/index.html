AWS ↔ Entra ID Workforce Identity Federation

This lab guides you through configuring workforce identity federation between AWS IAM Identity Center and Microsoft Entra ID using SAML and SCIM.

Cloud: AWS, Azure | Domain: Identity | Level: L4 | Estimated time: 60–90 minutes

Start Lab →

What You’ll Build

  • AWS IAM Identity Center integrated with Microsoft Entra ID
  • SAML-based authentication between AWS and Entra
  • SCIM user and group provisioning
  • Role-based access across cloud environments

Prerequisites

  • An AWS account with IAM Identity Center enabled
  • A Microsoft Entra ID tenant
  • Basic understanding of IAM and SAML concepts

Lab Steps

Follow the steps below to complete the lab. Each step builds on the previous one and mirrors real-world production workflows.

Architecture

Entra ID acts as the authoritative identity provider, while AWS IAM Identity Center functions as the service provider using SAML for authentication and SCIM for lifecycle management.

Security Controls

  • Centralized identity provider
  • Least-privilege permission sets
  • Just-in-time access patterns
  • Separation of authentication and authorization
