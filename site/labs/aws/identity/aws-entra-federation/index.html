AWS ↔ Entra ID Workforce Identity Federation

Implement workforce identity federation between AWS IAM Identity Center and Microsoft Entra ID using SAML and SCIM with least-privilege access.

AWS Entra ID Identity L4 Zero Trust

Lab Overview

This lab demonstrates architect-grade workforce identity federation between AWS and Microsoft Entra ID. It covers SAML trust establishment, SCIM provisioning, permission set design, and identity-centric access control aligned with Zero Trust principles.

What You Will Build

  • Federated authentication from Entra ID into AWS
  • Automated user and group provisioning via SCIM
  • Least-privilege AWS permission sets
  • Centralized identity governance

Lab Metadata

  • Level: L4 – Advanced
  • Cloud: AWS (Primary), Azure (Identity)
  • Domain: Identity & Access Management
  • Status: Live
  • Industry: Enterprise / Healthcare

Architecture

The architecture uses Entra ID as the authoritative identity provider and AWS IAM Identity Center as the service provider, with SAML for authentication and SCIM for lifecycle management.

Security Controls

  • Separation of authentication and authorization
