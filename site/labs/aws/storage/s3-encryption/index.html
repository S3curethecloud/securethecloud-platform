AWS S3 Encryption Basics

AWS Storage L1 30–45 mins

This lab explains how Amazon S3 protects data using encryption at rest and in transit, and walks through enabling and validating server-side encryption using AWS-managed keys.

Start Lab →

What You’ll Build

  • An S3 bucket with default encryption enabled
  • Encrypted objects using AWS-managed keys
  • Encrypted objects using customer-managed KMS keys
  • Policy-enforced encryption and audit evidence

Prerequisites

  • An AWS account with access to S3 and KMS
  • Basic familiarity with the AWS Console

Lab Steps

Step 1 — Understand What “Encryption” Means (Foundation)

Objective

Before touching AWS, you need to understand what you are protecting and why encryption matters.

Simple Explanation

Encryption is like locking your data in a box so only the right key can open it.

  • Files are called objects
  • Objects live inside buckets
  • Encryption prevents unauthorized reading

What You’ve Learned

  • S3 stores data in buckets
  • Encryption protects stored data
  • AWS can encrypt automatically

Step 2 — Create an S3 Bucket with Encryption Enabled

Objective

Create a secure bucket that encrypts all files automatically.

  1. Go to Amazon S3
  2. Create a bucket with a unique name
  3. Enable default encryption
  4. Select SSE-S3

What You’ve Achieved

  • Encrypted bucket created
  • No key management required

Step 3 — Upload a File and Verify Encryption

Objective

Confirm encryption is working.

  1. Upload a file
  2. Open file properties
  3. Verify SSE-S3 encryption

End of Step 3 — What You Have Achieved

  • Encrypted storage verified
  • Secure defaults confirmed

Step 4 — Upgrade to KMS Encryption (More Control)

Objective

Move from basic encryption to key-controlled encryption using AWS KMS.

Simple Explanation

AWS KMS lets you control who can use encryption keys, rotate them, and audit their use.

  1. Go to AWS KMS
  2. Create a symmetric encryption key
  3. Name it stc-s3-encryption-key
  4. Attach it to your S3 bucket using SSE-KMS

What You’ve Achieved

  • Centralized key control
  • Enterprise-grade encryption foundation

Step 5 — Enforce Encryption with a Bucket Policy

Objective

Prevent unencrypted uploads—even by mistake.

  1. Open the bucket Permissions tab
  2. Add a policy that denies unencrypted uploads
  3. Replace YOUR-BUCKET-NAME with your bucket name

What You’ve Achieved

  • Encryption is mandatory
  • Human error is blocked

Step 6 — Audit and Verify Encryption Activity

Objective

Prove encryption is active and auditable.

  1. Verify object encryption is SSE-KMS
  2. Review KMS key usage
  3. (Optional) Check CloudTrail encryption events

End of Step 6 — Lab Complete

  • KMS-controlled encryption enabled
  • Policy-enforced security
  • Audit-ready evidence
